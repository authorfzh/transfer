二级菜单

说明:

  • 该页面分为头部,菜单栏,ifream详情页
  • 页面分内部与外部的js,只需修改外部js即可,所需的数据json格式在下展示
  • 接口信息:
    (1)初始化加载的菜单
    需要封装头部和左侧菜单的json
    (2)点击事件方法
    1.头部菜单点击事件headMenuClick
    2.一级菜单点击事件openBar
    3.二级菜单点击事件multiMenuClick
    (3)路径方法
    由于ifream页面加载，其中的url 为项目的路径 ,所以需要定义获取项目路径的方法getContextPath

主节点 子节点 说明
Id 主键id

Icon 图标
url 跳转路径
Nodes 为菜单详情子信息
Id 主键
ParentId 父节点id
Text 节点名称
Icon 图标
url 跳转的路径

解释说明:头部菜单的json 在external.js外部js的 setHeardJson方法中返回, 没有菜单返回null

菜单json

[

主节点 子节点 说明
id 主键id
text 节点名称
Type 0为主菜单没有二级菜单，其他（0,或空）为树菜单 存在二级菜单
Icon 图标
url 跳转路径
TreeJson 二级菜单的树json

解释说明:一级主菜单 主要判断是否拥有二级菜单,没有url的地址为必填项。存在二级将二级菜单的json拼接完成赋予TreeJson值
